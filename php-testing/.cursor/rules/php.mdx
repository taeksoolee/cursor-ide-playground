# PHP Coding Guidelines

These are the PHP coding standards and practices to follow in this project.

- Use **PSR-12** coding standards.
- Always use **strict types**.
- Avoid using global variables.
- Prefer **dependency injection** over singleton or global access.
- Functions should be **pure** whenever possible.

- Use 4 spaces for indentation.
- Use short array syntax (`[]` instead of `array()`).
- Use `declare(strict_types=1);` at the top of every PHP file.
- Do not leave unused `use` statements.
- Avoid mixing HTML and PHP in the same file unless working in a templating context (e.g., Blade, Twig).

## ðŸ“¦ OOP Principles

- Always define visibility (`public`, `protected`, `private`) for properties and methods.
- Prefer **final** classes when extension is not required.
- Use **interfaces** to define contracts.
- Apply **SOLID** principles when designing classes.

use App\Repository\UserRepository;

    public function __construct(
